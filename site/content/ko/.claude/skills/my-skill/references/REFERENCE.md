# references/ - 상세 문서

이 디렉토리는 Claude가 더 깊은 컨텍스트가 필요할 때만 읽는 추가 문서를 포함합니다. 이것은 [점진적 공개](^처음에 적게 로드하고 필요할 때만 더 로드합니다. SKILL.md는 집중된 상태로 유지되고, 상세 자료는 여기에 위치합니다) 모델의 일부입니다.

## 여기에 넣을 것

- 상세 기술 레퍼런스 (API 사양, 프로토콜 문서)
- 도메인별 지식 (금융 규칙, 법적 요구사항)
- 확장된 예시와 엣지 케이스 문서
- 양식 템플릿 또는 구조화된 데이터 형식

## 작동 방식

1. SKILL.md가 이 파일을 참조합니다: "자세한 내용은 [references/REFERENCE.md](references/REFERENCE.md)를 참조하세요"
2. Claude는 먼저 SKILL.md를 읽습니다 (메인 지시사항)
3. Claude가 더 자세한 정보가 필요하면 링크를 따라 이 파일을 읽습니다
4. 이렇게 하면 컨텍스트 사용이 효율적입니다. 모든 것이 한 번에 로드되지 않습니다

## 모범 사례

- 개별 레퍼런스 파일은 하나의 주제에 집중하세요
- 작은 파일은 로드 시 컨텍스트 사용량이 적습니다
- 설명적인 파일 이름을 사용하세요: `api-spec.md`, `error-codes.md`, `migration-guide.md`
- SKILL.md에서 파일 참조는 한 단계 깊이로 유지하세요. 깊이 중첩된 체인은 피하세요
