# 스킬

Claude가 [필요할 때](^Claude는 시작 시 각 스킬의 이름과 설명만 로드합니다. 전체 내용은 해당 스킬이 현재 작업과 관련될 때만 로드됩니다) 발견하고 로드하는 재사용 가능한 지시사항, 참조 자료, 스크립트 패키지입니다. 스킬은 `SKILL.md` 파일을 포함하는 디렉토리입니다. 그것이 유일한 요구사항입니다.

## 빠른 시작

1. 폴더를 만드세요: `.claude/skills/my-skill/`
2. [frontmatter](^파일 상단 --- 마커 사이의 YAML 블록. 이름과 설명 같은 메타데이터를 포함합니다)에 name과 description을 가진 `SKILL.md`를 추가하세요
3. 본문에 지시사항을 작성하세요. 스킬이 활성화되면 Claude가 읽습니다
4. `/my-skill`로 호출하거나 관련성이 있을 때 Claude가 자동으로 로드하도록 하세요

스킬은 [Agent Skills 오픈 표준](https://agentskills.io)을 따르므로, Claude Code뿐만 아니라 여러 AI 도구에서 작동합니다.

## 작동 방식

스킬은 효율성을 위해 [점진적 공개](^컨텍스트를 효율적으로 관리하는 패턴: 처음에 적게 로드하고, 필요할 때만 더 로드합니다. Claude의 집중력을 유지하고 토큰 사용량을 줄입니다)를 사용합니다:

1. **발견**: 시작 시 Claude는 각 스킬의 `name`과 `description`만 로드합니다. 관련성을 판단하기에 충분한 정보입니다
2. **활성화**: 작업이 매칭되면 Claude는 전체 `SKILL.md`를 컨텍스트에 읽어들입니다
3. **실행**: Claude는 지시사항을 따르며, 선택적으로 참조 파일을 로드하거나 번들된 스크립트를 실행합니다

## 두 가지 유형의 스킬

| 유형 | 목적 | 예시 | 호출 방식 |
|---|---|---|---|
| 참조 | Claude가 작업에 적용하는 지식 | 스타일 가이드, 컨벤션, 도메인 규칙 | 관련성이 있을 때 자동 로드 |
| 작업 | 특정 동작을 위한 단계별 지시사항 | 배포, 스캐폴딩, 생성 | 보통 `/skill-name`으로 호출 |

## 스킬의 위치

| 위치 | 경로 | 범위 |
|---|---|---|
| 엔터프라이즈 | [관리형 설정](^IT/DevOps가 조직 차원의 정책을 위해 관리하는 시스템 수준 경로) | 조직의 모든 사용자 |
| 개인 | `~/.claude/skills/<name>/SKILL.md` | 모든 개인 프로젝트 |
| 프로젝트 | `.claude/skills/<name>/SKILL.md` | 이 프로젝트만 |
| 플러그인 | `<plugin>/skills/<name>/SKILL.md` | 플러그인이 활성화된 곳 |

여러 레벨에서 같은 이름의 스킬이 있으면, 더 높은 우선순위 위치가 우선합니다: 엔터프라이즈 > 개인 > 프로젝트.

## 커맨드와 스킬

[커스텀 커맨드](^슬래시 커맨드를 만드는 .claude/commands/의 마크다운 파일. 현재 스킬 아래로 통합됨)와 스킬은 이제 같은 시스템입니다. `.claude/commands/review.md`의 파일과 `.claude/skills/review/SKILL.md`의 스킬은 모두 `/review`를 생성하며 동일하게 작동합니다. 기존 커맨드 파일은 계속 작동합니다. 스킬은 선택적 기능을 추가합니다: 지원 파일, frontmatter 제어, 자동 로딩.

## 호출 제어

[frontmatter](^SKILL.md 상단 --- 마커 사이의 YAML 메타데이터) 설정으로 스킬을 트리거할 수 있는 주체를 제어하세요:

| 설정 | 사용자 호출 가능 | Claude 호출 가능 |
|---|---|---|
| (기본값) | 예 | 예 |
| `disable-model-invocation: true` | 예 | 아니오 |
| `user-invocable: false` | 아니오 | 예 |

부수 효과가 있는 동작(배포, 전송, 파괴적 작업)에는 `disable-model-invocation`을 사용하세요. 사용자가 직접 트리거하지 않아야 하는 백그라운드 지식에는 `user-invocable: false`를 사용하세요.

## 권한 제어

[권한 규칙](^settings.json 또는 .claude/settings.json에서 설정합니다. 자세한 내용은 설정 및 권한 섹션을 참조하세요)으로 Claude가 사용할 수 있는 스킬을 제한하세요:

- `Skill(commit)`: 특정 스킬 허용
- `Skill(deploy *)`: 스킬 접두사 거부
- `Skill` 도구 자체를 거부하여 모든 스킬 비활성화

## 번들 스킬

Claude Code와 함께 제공되며 모든 세션에서 사용 가능합니다:

| 커맨드 | 기능 |
|---|---|
| `/simplify` | 변경된 파일의 재사용성, 품질, 효율성을 검토하고 이슈를 수정합니다 |
| `/batch <instruction>` | 코드베이스 전체에 대규모 병렬 변경을 조율합니다 |
| `/debug [description]` | 디버그 로그를 읽어 세션을 문제 해결합니다 |
| `/claude-api` | 프로젝트 언어에 맞는 Claude API 레퍼런스를 로드합니다 |

## 팁

- 스킬 디렉토리 이름은 소문자와 하이픈, 최대 64자여야 합니다
- `SKILL.md`는 500줄 이하로 유지하세요. 상세 자료는 `references/` 또는 `assets/` 하위 디렉토리로 이동하세요
- `description` 필드는 Claude가 관련성을 판단하는 데 사용됩니다. 사용자가 자연스럽게 말할 키워드를 포함하세요
- 본문에 `$ARGUMENTS`를 사용하여 사용자 입력을 받으세요: `/my-skill some input here`
- [동적 컨텍스트 주입](^SKILL.md가 Claude에 도달하기 전에 실행되는 셸 명령어. 출력이 플레이스홀더를 대체합니다. 예: 변경된 파일 목록 주입)을 위해 `!` 백틱 구문을 사용하세요
- 자동 로딩에 의존하기 전에 `/skill-name`으로 직접 호출하여 스킬을 테스트하세요
