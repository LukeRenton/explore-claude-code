# 에이전트

에이전트는 특수 작업을 위해 생성하고 설정하는 격리된 Claude 인스턴스입니다. 두 가지 형태가 있습니다: 세션 내에서 작동하는 [서브에이전트](^세션 내에서 실행되어 작업을 수행하고 요약을 반환하는 집중된 워커)와, 별도의 세션 간에 협조하는 [에이전트 팀](^공유 작업 목록을 통해 협조하는 여러 독립적인 Claude Code 세션)입니다.

## 빠른 시작

1. Claude Code 내에서 `/agents`를 실행하여 대화형 에이전트 관리자를 엽니다
2. **Create new agent**를 선택한 다음 범위(프로젝트 또는 사용자)를 선택합니다
3. **Generate with Claude**를 사용하여 에이전트가 무엇을 해야 하는지 설명합니다
4. 에이전트가 접근할 수 있는 도구를 선택하고, 모델을 골라 저장합니다

에이전트를 즉시 사용할 수 있습니다. 시도해 보세요: "Use the my-agent agent to review this module."

## 서브에이전트 작동 방식

서브에이전트는 설정을 위한 YAML frontmatter와 시스템 프롬프트가 되는 Markdown 본문이 있는 `.md` 파일입니다. 각 서브에이전트는 자체 컨텍스트 윈도우에서 실행되며, 허용된 도구만 사용하고, 메인 대화에 요약을 반환합니다.

1. Claude가 요청을 서브에이전트의 `description` 필드와 매칭합니다
2. 서브에이전트의 시스템 프롬프트로 새 컨텍스트 윈도우가 생성됩니다
3. 서브에이전트가 독립적으로 작업합니다 (파일 읽기, 명령어 실행 등)
4. 결과가 메인 대화로 요약되어 반환됩니다

서브에이전트는 포그라운드(블로킹) 또는 백그라운드(동시)에서 실행할 수 있습니다. `Ctrl+B`를 눌러 실행 중인 작업을 백그라운드로 보내세요.

## 에이전트 위치

| 위치 | 범위 | 우선순위 |
|----------|-------|----------|
| `--agents` CLI 플래그 | 현재 세션만 | 1 (최고) |
| `.claude/agents/` | 이 프로젝트 (버전 관리에 체크인) | 2 |
| `~/.claude/agents/` | 모든 개인 프로젝트 | 3 |
| 플러그인 `agents/` 디렉토리 | 플러그인이 활성화된 곳 | 4 (최저) |

여러 에이전트가 같은 이름을 공유하면 더 높은 우선순위의 위치가 우선합니다.

## 내장 서브에이전트

Claude Code에는 자동으로 사용하는 서브에이전트가 포함되어 있습니다:

- **Explore**: 빠른 읽기 전용 코드베이스 검색 (Haiku에서 실행)
- **Plan**: 구조화된 계획 및 분석
- **General-purpose**: 기타 위임을 위한 유연한 도우미

같은 이름으로 더 높은 우선순위 범위에 커스텀 에이전트를 만들어 내장 에이전트를 오버라이드할 수 있습니다.

## Frontmatter 레퍼런스

| 필드 | 필수 | 목적 |
|-------|----------|---------|
| `name` | 예 | 고유 식별자 (소문자, 하이픈) |
| `description` | 예 | Claude가 이 에이전트에 위임할 시점을 판단하는 설명 |
| `tools` | 아니오 | 허용할 도구 목록 (생략 시 모두 상속) |
| `disallowedTools` | 아니오 | 상속된 도구에서 제거할 거부 목록 |
| `model` | 아니오 | `sonnet`, `opus`, `haiku`, 또는 `inherit` (기본값) |
| `permissionMode` | 아니오 | `default`, `acceptEdits`, `dontAsk`, `bypassPermissions`, `plan` |
| `maxTurns` | 아니오 | 에이전트가 중지되기 전 최대 에이전트 턴 수 |
| `skills` | 아니오 | 시작 시 에이전트 컨텍스트에 주입할 스킬 |
| `mcpServers` | 아니오 | 이 에이전트에서 사용 가능한 MCP 서버 |
| `hooks` | 아니오 | 이 에이전트에 범위가 지정된 생명주기 훅 |
| `memory` | 아니오 | 영구 메모리 범위: `user`, `project`, 또는 `local` |
| `background` | 아니오 | `true`로 설정하면 항상 백그라운드 작업으로 실행 |
| `isolation` | 아니오 | `worktree`로 설정하면 임시 git worktree에서 실행 |

## 에이전트 팀

에이전트 팀은 병렬로 작업하는 여러 독립적인 Claude Code 세션을 조율합니다. 하나의 세션이 리드 역할을 하고, 나머지는 자체 컨텍스트 윈도우를 가진 팀원입니다.

에이전트 팀은 실험적 기능입니다. `settings.json`에서 활성화하세요:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

그런 다음 팀 구조를 자연어로 설명합니다:

```
Create an agent team to review PR #142. Spawn three reviewers:
one focused on security, one checking performance, one validating tests.
```

### 서브에이전트 vs 에이전트 팀

| | 서브에이전트 | 에이전트 팀 |
|---|-----------|-------------|
| 커뮤니케이션 | 호출자에게만 결과 보고 | 팀원이 서로 직접 메시지 |
| 컨텍스트 | 자체 윈도우; 결과가 메인 세션으로 반환 | 자체 윈도우; 완전 독립 |
| 조율 | 메인 에이전트가 모든 작업 관리 | 공유 작업 목록으로 자체 조율 |
| 적합한 용도 | 결과만 중요한 집중 작업 | 논의가 필요한 복잡한 작업 |
| 토큰 비용 | 낮음 (결과가 요약되어 반환) | 높음 (각 팀원이 별도 인스턴스) |

빠르고 집중된 워커에는 서브에이전트를 사용하세요. 팀원이 발견 사항을 공유하고 스스로 조율해야 할 때는 에이전트 팀을 사용하세요.

## 일반적인 패턴

- **장황한 출력 격리**: 테스트 실행이나 로그 분석을 서브에이전트에 위임하여 메인 컨텍스트에서 출력을 제외합니다
- **병렬 조사**: 여러 서브에이전트를 생성하여 코드베이스의 다른 부분을 동시에 조사합니다
- **서브에이전트 체이닝**: 하나는 분석용, 그 결과를 다른 하나에 전달하여 구현합니다
- **경쟁 가설**: 에이전트 팀 멤버를 생성하여 다른 이론을 조사하고 서로 토론합니다

## 팁

- 상세한 `description` 필드를 작성하여 Claude가 언제 위임할지 알 수 있게 하세요. 자동으로 사용하길 원하는 에이전트에는 "use proactively"를 포함하세요
- 에이전트에 필요한 도구만 부여하세요. 읽기 전용 리뷰어에게 Write나 Edit를 주지 마세요
- 서브에이전트는 다른 서브에이전트를 생성할 수 없습니다. 중첩 위임이 필요하면 스킬을 사용하거나 메인 대화에서 체이닝하세요
- `memory: user`를 사용하여 에이전트가 세션 간에 지식을 축적할 수 있게 하세요. 시작 전에 "check your memory"를, 완료 후에 "save what you learned"를 요청하세요
- 에이전트 팀은 3-5명의 팀원과 각 5-6개 작업이 가장 효과적입니다. 팀원이 많을수록 조율 오버헤드가 증가합니다
- 팀원은 작업 디렉토리의 `CLAUDE.md`를 로드하므로 프로젝트 컨벤션이 팀 전체에 적용됩니다
