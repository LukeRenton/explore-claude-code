1. **정확성**: 로직이 의도한 대로 작동하는가?
2. **보안**: 인젝션 위험, 유출된 시크릿, 인증 우회가 있는가?
3. **테스트**: 엣지 케이스가 커버되는가?
4. **스타일**: CLAUDE.md의 컨벤션을 따르는가?

발견된 각 이슈를 다음과 같이 분류하세요:
- **반드시 수정**: 버그, 보안 이슈
- **수정 권장**: 누락된 테스트, 불명확한 네이밍
- **참고**: 스타일 선호, 선택적 개선사항

먼저 `git diff main...HEAD`를 실행하여 모든 변경사항을 확인하세요.

$ARGUMENTS

---

이것은 스타터 커맨드 파일입니다. 파일 이름 `my-command.md`가 슬래시 커맨드 `/my-command`가 됩니다.

`---` 위의 모든 내용은 커맨드를 호출할 때 Claude가 받는 프롬프트입니다. `$ARGUMENTS`는 커맨드 이름 뒤에 입력하는 내용으로 대체됩니다.

예를 들어, `/my-command focus on auth`는 `$ARGUMENTS`를 "focus on auth"로 대체합니다.

## 좋은 커맨드의 조건

- 명확하고 구체적인 작업 (개방형이 아닌)
- 프로젝트의 표준이나 컨벤션에 대한 컨텍스트
- 결과가 일관되도록 구조화된 출력 형식
- 호출마다 커스터마이징할 수 있는 `$ARGUMENTS`

## 제한사항

커맨드는 단순한 프롬프트 템플릿입니다. 다음을 지원하지 않습니다:

- Frontmatter (모델 오버라이드, 호출 제어, 도구 제한)
- 지원 파일 (scripts, references, assets)
- 동적 컨텍스트 주입 (`!` 백틱 구문)
- 작업 관련성에 따른 자동 로딩

이러한 기능이 필요하면 [스킬](^스킬은 커맨드의 더 새롭고 강력한 대체 시스템입니다. 전체 구조는 skills/ 디렉토리를 참조하세요)을 대신 사용하세요. 커맨드와 스킬은 동일한 `/slash-command` 인터페이스를 생성합니다.
